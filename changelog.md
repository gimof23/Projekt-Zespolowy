# Changelog

- Backend poukładany na wzorzec MVC: osobno trasy, osobno logika obsługi stron i zapytań, zamiast jednego wielkiego pliku startowego z wszystkim w środku.
- Wspólne sprawdzanie kto jest zalogowany, adminem czy pracownikiem jest w jednym miejscu, żeby nie powtarzać tego samego w kółko.
- Nazwa kina w aplikacji to CinemaŚty zamiast Cheljoz, żeby było spójnie z brandingiem.
- Repertuar i API do kalendarza seansów ogarnięte tak, żeby nie było duplikatów ani literówek w widokach.
- Rezerwacja miejsc: kolejność zapisu rezerwacji i otwierania strony z salą jest poprawna technicznie.
- Panel pracownika: czytelniejsza tabela seansów, wygodniejsze zarządzanie biletami w oknie modalnym, w tym przełączanie stron gdy jest dużo pozycji.
